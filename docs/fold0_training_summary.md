## 7. 错误模式分析

### 7.1 体素级错误分布

```
Precision = 0.5819  (预测为正样本中，58% 是正确的)
Recall    = 0.3443  (实际正样本中，仅34% 被找回)

结论: 漏检 (False Negative) 比误检 (False Positive) 更严重
      模型主要问题是"找不到病灶"，而非"到处乱画"
```

### 7.2 BPH 失败原因推断

1. **影像表型差异**: BPH 与 PCA 病灶在影像上表现不同
2. **特征学习偏向**: 模型更容易学到 PCA 的判别特征
3. **标签统一问题**: 将不同病理类型的 ROI 统一标记为 lesion=1 可能掩盖了本质差异

## 9. 后续优化建议

### 9.1 立即执行 (高优先级)

1. **亚群分开报告指标**
   - 后续各折都按 BPH / PCA 分别统计 Dice、Recall、Precision
   - 建立亚群性能追踪表

2. **复核 BPH 失败病例**
   - 检查 BPH 标签风格是否与 PCA 一致
   - 确认病灶定义在两个亚群中是否相同
   - 分析 BPH 影像表型与 PCA 的根本差异

### 9.2 中期优化 (中优先级)

3. **考虑任务重新定义**
   - **方案 A**: BPH 和 PCA 分开建模 (两个独立模型)
   - **方案 B**: 改为多类标签 (lesion_BPH=1, lesion_PCA=2)
   - **方案 C**: 如果必须统一，考虑以下技术改进

4. **技术改进方向** (如坚持统一任务)
   - **采样策略**: lesion-aware sampling，增加 BPH 样本权重
   - **损失函数**: Tversky Loss (强调 Recall) 或 Focal Loss
   - **级联策略**: 先检测 ROI，再精细分割
   - **数据增强**: 针对 BPH 病灶特点的专项增强

### 9.3 不推荐的方向

| 不推荐 | 原因 |
|--------|------|
| 继续增加 epochs | epoch 62 已达平台期，延长无益 |
| 单纯调学习率 | 训练稳定，不是优化器问题 |
| 换不同 backbone | 当前模型能学好 PCA，说明 capacity 足够 |
